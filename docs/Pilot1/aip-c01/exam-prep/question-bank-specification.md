# Question Bank Specification

## Purpose

The Pilot1 question bank supports AIP-C01 exam readiness. It stores reviewed
practice items that are traceable to the official exam guide and to curriculum
units.

Raw LLM output, crawled questions, or manually drafted prompts are not approved
question-bank content until reviewed.

## Provenance Rule

Raw candidate files must declare their true provenance before review. Use
provider/model metadata only for content actually generated by that
provider/model. Manual, Codex-authored, reviewer-authored, imported, or
transformed candidates must be labeled as such.

Review approval is separate from provenance. A manually authored item can be
approved if it passes review, and an LLM-generated item can be culled. The
reviewed bank must preserve enough raw-source information to audit both the
item's origin and the review decision.

## Source Extraction Rule

Before objective-tagged generation or review, use
`docs/Pilot1/aip-c01/source-material/ai-professional-01.objectives.json` as the
canonical source for official domain, task, Skill X.Y.Z statements, and domain
weights. Use the extracted Markdown transcript for human wording checks and
the PDF only as the source of record for audit or regeneration.

Confirm that the objective JSON exposes the deepest level used by the bank
schema. For AIP-C01, that means official domain, task, and Skill X.Y.Z
statements when `exam_skill` is populated.

Record the source file, extraction method or tool, and any known gaps from
`ai-professional-01.extraction-meta.json`. Do not replace official skill
statements with local paraphrases unless the field is explicitly labeled
editorial, such as `exam_skill_local`.

## Bank Completion Rule

A day's reviewed bank can contain a smaller approved slice for early use, but
it does not count as complete until it has at least `10` approved items for
each curriculum-order topic covered that day.

For example, a day with 10 curriculum-order topics needs at least 100 approved
items before the day's bank is complete. A smaller set must be labeled as a
minimum viable approved slice, pilot slice, or partial bank.

## Continuous Improvement Rule

Every review pass must produce a cull summary before additional generation.
The summary should group rejected items by failure reason, such as
`not-scenario-based`, `recall-only`, `keyword-trivia`, `duplicate-pattern`,
`schema-drift`, `stale-claim`, or `weak-distractors`.

Every culled item must also include audit evidence explaining the rejection.
The evidence should cite the concrete field value, stem pattern, missing schema
field, matched stale-claim phrase, source-check issue, or other observable fact
that caused the rejection. Rejection labels alone are not sufficient.

Review checks must inspect the full item text, including stems, options,
correct rationales, distractor rationales, misconception tags, source traces,
and remediation targets. Distractor rationales are teaching content; a false
or stale claim there can fail an item even when the keyed answer is defensible.

After review, scan the approved bank for the same known-bad patterns that the
reviewer uses to cull items. This negative-control check must cover stale AWS
service claims, unsupported date-bound phrases, duplicate stems, schema drift,
missing traceability fields, and provenance defects. A day's bank is not
complete if a disqualifying pattern survives in approved content.

Before regenerating or topping up a bank, update the generation prompt or
top-up prompt with:

- the highest-frequency cull reasons;
- examples of rejected stem patterns to avoid;
- existing approved stems and rejected stem excerpts for the same topic;
- the minimum number of extra raw candidates needed to survive expected culls;
- any service claims that must be source-checked against current official AWS
  documentation.

A prompt that ignores the prior cull summary is not ready for another
generation pass.

Prompt guidance is not sufficient by itself. The review pass must also reject
exact or near-duplicate stems within the same curriculum-order topic and log
them as `duplicate-pattern` culls. A duplicate cull must include the rejected
stem, the matched approved stem, the matched item source or ID, and whether the
match was exact or near-duplicate.

## Required Item Fields

| Field | Requirement |
|---|---|
| `item_id` | Stable identifier, such as `P1-AIP-D1-T15-001`. |
| `status` | `draft`, `reviewed`, `approved`, `retired`, or `needs-source-check`. |
| `official_exam_code` | `AIP-C01`. |
| `exam_domain` | Official domain number and name. |
| `exam_task` | Official task statement. |
| `exam_skill` | Official Skill X.Y.Z statement from the exam guide when the item maps to a skill-level objective. |
| `exam_skill_local` | Optional reviewer-written narrower focus when the item tests a smaller distinction inside the official skill. |
| `learning_unit` | Curriculum unit ID when units exist. |
| `accelerated_day` | Day 1 through Day 7 in the accelerated path. |
| `knowledge_type` | Dominant knowledge type tested. |
| `question_format` | `multiple-choice` or `multiple-response`. |
| `difficulty` | `easy`, `medium`, `hard`, or `exam-plus`. |
| `cognitive_demand` | Recall, explain, implement, integrate, configure, select, diagnose, optimize, compare, or judge. |
| `stem` | Question prompt or scenario. |
| `options` | Answer options. |
| `correct_answer` | Correct option or options. |
| `rationale_correct` | Why the correct answer is correct. |
| `rationale_distractors` | Why each distractor is wrong or less appropriate. |
| `misconception_tags` | Misconception, trap, or tradeoff tested. |
| `source_trace` | Official AWS source or trusted project source supporting the claim. |
| `remediation_target` | Unit, section, lab, or remediation route. |
| `estimated_time_seconds` | Expected time for a prepared learner. |
| `last_reviewed` | Date the item was last checked. |

## Simulated Exam Rules

Full simulated exams should:

- use 75 questions;
- use a 180-minute timer;
- approximate official domain weighting;
- use only multiple-choice and multiple-response questions;
- include no copied official AWS practice questions, proprietary exam dumps, or
  third-party copyrighted question-bank content;
- produce a domain, task, confidence, and remediation report.

## Review Checklist

Before approval, confirm:

- the item maps to AIP-C01, not another AWS exam;
- the item is professional-level and scenario-based when appropriate;
- the item is not a duplicate or near-duplicate of an already approved item for
  the same curriculum-order topic;
- the correct answer is unambiguously best;
- distractors are plausible but defensibly wrong;
- the explanation teaches the relevant distinction;
- technical claims can be checked against official AWS sources;
- remediation points to a specific learning resource.
