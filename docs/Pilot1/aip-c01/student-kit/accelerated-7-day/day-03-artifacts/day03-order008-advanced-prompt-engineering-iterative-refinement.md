## Purpose

Improve prompts through controlled experiments, not random edits. A strong
prompt change should be tied to a hypothesized failure mode and measured
against examples.

## Mastery Evidence

You can connect a prompt change to a hypothesis, test it against examples, and
explain both the improvement and the new risk introduced by the change.

## Remediation

If your prompt edits are just successive rewrites, add a hypothesis and a test
case to each version. Keep only changes whose effect can be observed in the
test results.
