# Pilot Runbook: Solo Student, Teacher, And Examiner

## Purpose

Use this runbook if you are piloting the accelerated AIP-C01 path by yourself.
It explains how to run one day of the system while wearing all three hats:
student, teacher, and examiner.

## The Core Rule

Do not judge while you are attempting.

Separate the work into three passes:

1. **Student pass:** attempt the artifact honestly.
2. **Teacher pass:** diagnose gaps and choose remediation.
3. **Examiner pass:** decide whether the evidence is good enough.

This separation prevents two common pilot problems:

- the student gives up too early because the examiner voice is too loud;
- the examiner goes soft because the student worked hard.

## One-Day Solo Workflow

### Step 1: Set The Day Intention

Write this at the top of your working note:

```text
Day:
Today's artifact:
Today I must be able to:
The most likely gap is:
I will stop only after:
```

### Step 2: Student Pass

Fill the artifact without trying to be perfect.

Rules:

- use your current understanding first;
- mark assumptions;
- mark confidence for hard answers;
- leave a visible `?` where you are unsure;
- do not ask AI to fill the answer.

### Step 3: Teacher Pass

Review your own answer as a teacher.

For every weak section, identify the gap type:

| Gap type | Teacher question |
|---|---|
| Recall | What term, service, feature, or fact is missing? |
| Conceptual | What relationship is misunderstood? |
| Procedural | What step or workflow cannot be performed? |
| Conditional | What constraint changes the correct choice? |
| Causal | What mechanism explains the outcome? |
| Diagnostic | What evidence should be checked before fixing? |
| Governance | Who owns it, what control applies, what evidence proves it? |
| Metacognitive | Was confidence aligned with performance? |

Then choose one remediation action. Keep it small and concrete.

### Step 4: Examiner Pass

Score the artifact section by section.

| Score | Meaning |
|---:|---|
| 0 | Missing or generic. |
| 1 | Relevant terms, weak application. |
| 2 | Usable answer with rationale. |
| 3 | Strong answer with tradeoffs, rejected alternatives, evidence, and risk. |

Record:

```text
Sections scored 0:
Sections scored 1:
Strongest section:
Weakest section:
Severe gaps:
Pass / remediate / pause:
```

### Step 5: Timed Practice

Answer the daily practice set under time pressure.

For every missed, guessed, or weak item, record:

```text
Question/topic:
My answer:
Correct answer:
Why the correct answer is right:
Why the tempting wrong answer is wrong:
Gap type:
Remediation:
```

### Step 6: End-Of-Day Decision

Choose one:

- `Pass day`: artifact usable, practice complete, severe gaps absent or
  remediated.
- `Pass with carryover`: artifact usable, but named gaps must be revisited.
- `Do not move on yet`: critical artifact or severe knowledge gaps remain.

## Pilot Feedback Log

At the end of each day, write 5-10 bullet points:

- What instruction was unclear?
- Which artifact section helped most?
- Which section felt like paperwork?
- Where did you need an example?
- Where did the workload feel too heavy or too light?
- What did you want from a teacher?
- What did you want from an examiner?
- What should change before another learner uses this?

These notes are part of the pilot. They improve the system.
